Frequently asked questions

HTML vs HTML5

HTML, of HyperText Markup Language, is de standaardtaal voor het maken van webpagina's. Het bestaat uit tags en elementen die de structuur en inhoud van een webpagina definiëren. HTML wordt gebruikt om tekst, afbeeldingen, links en meer weer te geven.

Om een hyperlink toe te voegen, gebruik je het -element en de href-attribuut. Bijvoorbeeld: Ga naar voorbeeldwebsite. Dit maakt een link met de tekst "Ga naar voorbeeldwebsite".

HTML (HyperText Markup Language) verwijst naar de oorspronkelijke versie van de webtaal. HTML5 is de nieuwere, meer geavanceerde versie. Een belangrijk verschil is dat HTML5 uitgebreide ondersteuning biedt voor multimedia, zoals video- en audioweergave, terwijl HTML beperkter is op dit gebied. HTML5 introduceert ook nieuwe elementen en attributen voor betere semantische structuur en interactieve functionaliteit, zoals video, audio, canvas, article, nav, en meer. HTML5 heeft de webstandaard geëvolueerd en biedt ontwikkelaars meer krachtige tools voor het bouwen van moderne webpagina's.

SASS vs CSS

CSS: CSS-code kan repetitief en minder leesbaar zijn. SASS/SCSS: SASS/SCSS maakt gebruik van variabelen en nesting om de leesbaarheid van de code te verbeteren, waardoor het gemakkelijker wordt om complexe stijlen te beheren.

CSS: CSS heeft beperkte herbruikbaarheid, wat kan leiden tot duplicatie van code. SASS/SCSS: SASS/SCSS stelt mixins en functies beschikbaar, waardoor ontwikkelaars code gemakkelijk kunnen hergebruiken en onderhouden. Dit bevordert consistentie in stijlen en minimaliseert herhaling.

CSS: CSS mist ingebouwde ondersteuning voor variabelen. SASS/SCSS: SASS/SCSS introduceert variabelen die ontwikkelaars kunnen gebruiken om waarden op één plek op te slaan en ze vervolgens overal in de stijlen te hergebruiken. Dit vereenvoudigt het beheer van kleuren, lettertypen en andere stijlgerelateerde waarden.

CSS: CSS heeft geen ingebouwde nesting van regels, waardoor code soms ongestructureerd kan lijken. SASS/SCSS: SASS/SCSS maakt het mogelijk om regels te nesten, wat de structuur van de stijlen verbetert en de relatie tussen elementen op een webpagina duidelijker maakt. Dit leidt tot een betere onderhoudbaarheid en begrijpelijkheid van de stijlen.

CSS: CSS-imports leiden tot meerdere HTTP-verzoeken wanneer afzonderlijke bestanden worden geïmporteerd. SASS/SCSS: SASS/SCSS kan meerdere bestanden samenvoegen in één CSS-uitvoer, wat de prestaties kan verbeteren door het aantal HTTP-verzoeken te verminderen. Dit maakt het eenvoudiger om code modulair te organiseren en te beheren.
